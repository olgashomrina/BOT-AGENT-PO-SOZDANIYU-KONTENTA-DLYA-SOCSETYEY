# Спецификация проекта: ИИ-агент для контента в соцсетях

Дата: 2026-07-17
Статус: актуальная спецификация для разработки. Заменяет собой скоуп из `technical-specification.md` там, где он расходится с решениями ниже (тот файл — исходное ТЗ до критики и рыночной проверки, этот — то, что реально идёт в разработку).
Источники: `idea.md`, `technical-specification.md`, `user-scenarios.md`, `research.md`, `idea-critique-and-recommendations.md`, `china-vietnam-messengers-research.md`, `docs/superpowers/specs/2026-07-16-mvp1-telegram-bot-design.md`, `docs/superpowers/specs/2026-07-17-premium-consultant-vision.md`.

## 1. Боль и позиционирование

Ядро: снять ступор перед пустым экраном — превратить ссылку/голосовое сообщение в готовый пакет текстов для соцсетей за секунды, в стиле автора.

**Рыночная поправка (из идеи-критики, Часть 4):** боль реальна и массова, но рынок насыщен — 20-30+ прямых конкурентов только на русском рынке, включая почти идентичный ChatPost («Геннадий»), который уже делает генерацию из ссылки/голоса/фото в стиле пользователя + анализ 50 последних постов конкурентов. **«Мы первые» — не стратегия.** Дифференциатор — не список функций (все три «уникальные» фичи идеи уже мейнстрим у конкурентов), а: глубина исполнения (содержательный анализ стиля, а не поверхностный тон — см. «Taplio-эффект»), память и проактивность (этого не нашлось ни у одного проверенного конкурента), и/или узкая ниша аудитории.

## 2. Аудитория

- **Запуск:** владелец продукта и несколько знакомых — не публичный релиз.
- **Дальше:** маркетологи, предприниматели, SMM-специалисты, эксперты/консультанты, которым нужен быстрый и персонализированный контент для соцсетей.
- Команды и агентства — не аудитория MVP-1, это аудитория Agency-уровня (см. раздел 8).

## 3. Скоуп MVP-1 — что входит

MVP-1 = **только текстовое ядро**: ссылка/голос → расшифровка → несколько вариантов текста для Telegram и VK. Ничего больше.

**Входные форматы:**
- Ссылка на статью/материал.
- Голосовое сообщение или аудио-файл в Telegram.
- Обычный текст.

**Обработка:**
- Голос → расшифровка через AI Gateway, **пользователь видит и может поправить текст перед генерацией** (снижает и ошибки распознавания, и риск сгенерировать контент из неверно понятой мысли).
- Ссылка → извлечение статьи через `trafilatura`.
- Текст → используется как есть.

**Генерация:**
- Несколько вариантов текста на выбор, отдельно под Telegram и VK.
- Разметка, эмодзи, базовый CTA.
- Кнопки доработки уже сгенерированного варианта («Ещё вариант» / «Короче») — дешёвый, проверенный UX-паттерн из research.md (аналог команд Typefully).

**Локализация:**
- Интерфейс бота — на 4 языках: RU/EN/VI/ZH, команда `/language`, по умолчанию — `language_code` Telegram.
- Язык генерации контента — независимая настройка от языка интерфейса.
- **Важно:** локализация интерфейса ≠ выход на рынки Китая/Вьетнама. Telegram заблокирован в обеих странах (Китай — с 2015, Вьетнам — с мая 2025, статус на середину 2026 не подтверждён как снятый — см. `china-vietnam-messengers-research.md`). Реальная аудитория для этих языков — диаспора и пользователи с VPN, не массовый локальный рынок. Продукт не должен заявлять доступ к этим рынкам как достигнутый.

## 4. Явно НЕ входит в MVP-1

| Функция | Статус | Причина |
|---|---|---|
| Веб-приложение с личным кабинетом | Перенесено на **Pro-уровень** | Второй полноценный фронтенд/бэкенд-контур не нужен, пока продукт не вышел за пределы «владелец + знакомые» |
| Прямая автопубликация в несколько соцсетей | Перенесено на **Pro-уровень** | OAuth/бизнес-верификация на каждую площадку не окупается на этапе теста |
| Командное согласование/коллаборация | Перенесено на **Agency-уровень** | Аудитория запуска — соло-пользователи, этой проблемы у них ещё нет |
| Парсинг лидов и самопродвижение бота в чужих чатах | **Исключено полностью**, в любом виде и на любом уровне | Юридически рискованно (152-ФЗ, правила Telegram) и, по данным исследования (Часть 5 критики), активно осуждается профессиональным сообществом («нейрокомментинг») — прямая угроза репутации до появления первых пользователей |
| Лидогенерация для бизнеса клиента (легальная версия) | Перенесено на **Agency-уровень**, переосмыслено | Приемлемо только как платный инструмент с человеком в контуре, не как самопродвижение бота |
| Рекомендации сторонних инструментов для генерации фото/видео | Исключено из ближайшего скоупа | Нет собственного УТП, это редирект на чужие продукты, требует постоянной поддержки актуальности |
| Наложение музыки и текста на видео | Исключено из ближайшего скоупа | Отдельный технический домен (видеообработка, лицензии на музыку), не связан с текстовым ядром |
| Опросники/тесты/доп. форматы контента | Отложено | Не вредно, но размывает фокус, пока не отполировано ядро |
| Полноценная память (семантический поиск по истории) | Перенесено на **Pro-уровень** | Отдельный архитектурный проект (эмбеддинги, векторный поиск) — несопоставим по сложности со счётчиком запросов в SQLite MVP-1 |
| Фоновый мониторинг конкурентов | Перенесено на **Agency-уровень** | Принципиально другая по стоимости и сложности система, чем разовый анализ (см. раздел 8) |

## 5. Архитектура и стек (MVP-1)

- **Язык/фреймворк:** Python + aiogram, режим long polling (без домена/SSL).
- **Процесс:** автоперезапуск (systemd/Docker restart policy) + уведомление владельцу в личку при падении бота.
- **ИИ:** единый российский прокси-сервис (типа vsegpt.ru) — доступ к Claude/GPT-класс моделям и к Whisper, оплата в рублях.
- **AI Gateway:** все вызовы к ИИ-провайдеру спрятаны за внутренним интерфейсом (`generate_text()`, `transcribe()`) — замена провайдера в будущем должна быть заменой конфигурации, а не переписыванием бота.
- **Извлечение текста по ссылке:** `trafilatura` — работает с кириллицей, вьетнамским (латиница с диакритикой) и китайским.
- **Хранилище:** SQLite — белый список Telegram ID, языковая настройка пользователя, счётчик запросов с дневным/месячным лимитом на человека. Полноценная история контента — вне MVP-1.
- **Хостинг:** Timeweb Cloud (российский облачный провайдер, оплата картой РФ).
- **Секреты:** токен бота и ключ ИИ-прокси — в `.env`, не в репозитории.

**Компоненты:**
1. **Bot Handler** — приём сообщений, проверка белого списка и лимита запросов, определение типа входа.
2. **Input Processor** — голос → расшифровка с показом пользователю для проверки/правки; ссылка → извлечение статьи; текст → как есть.
3. **AI Gateway** — единая точка вызова ИИ-прокси.
4. **Content Generator** — промпт с учётом целевых площадок и языка контента → несколько вариантов текста.
5. **Localization** — словари строк интерфейса на 4 языках.
6. **Output Formatter** — форматирование и отправка вариантов с инлайн-кнопками доработки.

## 6. Известные ограничения и риски (зафиксированные)

- Telegram заблокирован в материковом Китае и во Вьетнаме — язык интерфейса не равен доступу к рынку (см. раздел 3).
- Распознавание вьетнамской речи (Whisper) даёт заметно больше ошибок — митигировано показом расшифровки пользователю перед генерацией.
- Импорт стиля из чужих публичных профилей (Instagram и др.) как источника для анализа стиля — потенциальный конфликт с ToS площадок при автоматизированном сборе; для MVP-1 предпочтительнее ручная загрузка пользователем своих примеров, автоматический импорт — открытый вопрос на будущее (см. раздел 9).
- Нет модерации сгенерированного текста перед выдачей — риск для продающих текстов/офферов, публикуемых от имени бренда пользователя; минимальная проверка на явно токсичный/недостоверный контент должна быть заложена до того, как в продукте появятся продающие форматы.
- Нет плана резервного копирования SQLite на Timeweb — некритично, пока в базе только счётчики и белый список, но обязательно нужно решить до появления истории контента (Pro-уровень).
- Стоимость ИИ-вызовов в MVP-1 контролируется только лимитом запросов на пользователя — этого достаточно, пока все вызовы происходят по прямому запросу; при появлении фоновых функций (память, мониторинг конкурентов) нужен отдельный бюджетный контроль до их запуска, не после.

## 7. Метрики успеха MVP-1

Явно не были зафиксированы нигде ранее (пробел, отмеченный в критике) — предлагаемые ориентиры, требуют подтверждения владельцем:
- Владелец и тестовая группа знакомых пользуются ботом регулярно (например, не менее N раз в неделю) в течение согласованного периода после запуска.
- Доля сгенерированных вариантов, которые публикуются без существенной правки — качественный сигнал того, что стиль/формат действительно экономит время, а не создаёт новую работу по переписыванию.
- Отсутствие критических сбоев (падений бота без автоперезапуска, потери сообщений) за тестовый период.

## 8. Дорожная карта после MVP-1

1. **MVP-1** (в разработке) — текстовое ядро, как описано выше.
2. **Анализ стиля пользователя** — по критике (Taplio-эффект) должен быть содержательным: конкретные обороты, длина предложений, частотность эмодзи — а не общее описание тона. Ручная загрузка примеров в первую очередь, автоимпорт из профилей — отдельное решение.
3. **Разовый анализ конкурентов** — по явному запросу пользователя, привязан к конкретному посту/теме. Не фоновый мониторинг (это Agency-уровень, см. ниже) — версии принципиально разной сложности и стоимости, разово выбрана эта как ближайшая.
4. **Фото/видео → посты/подписи/хэштеги** по загруженным медиа. Без наложения музыки/текста на видео (см. раздел 4).
5. **Pro-уровень** (см. `docs/superpowers/specs/2026-07-17-premium-consultant-vision.md`): полноценная память с семантическим поиском по истории, веб-кабинет, контент-календарь, обратная связь по эффективности постов, автопубликация, проактивный еженедельный дайджест.
   - **Обновление 2026-07-20:** по итогам реальной обратной связи от тестовой группы (Фаза 11 `Plan.md`) автопубликация в Telegram-канал вынесена из общего пункта 5 и делается раньше остального Pro-уровня, отдельной Фазой 12 в `Plan.md` — как самая быстрая и не требующая новой инфраструктуры (не нужен OAuth/домен). VK, Instagram, B17 и другие площадки остаются в общем Pro-уровне из-за существенно большей сложности (OAuth, верификация Meta, неизвестный статус API у нишевых сайтов).
6. **Agency-уровень**: мультиклиентность, командное согласование, фоновый мониторинг конкурентов, white-label отчётность, легальная лидогенерация для бизнеса клиента, интеграции с CRM.

## 9. Открытые вопросы

- Формат вывода и детали UX кнопок доработки текста — не согласованы в деталях.
- Поток данных и обработка ошибок (что показывать пользователю при сбое извлечения ссылки/транскрибации) — не согласованы.
- План тестирования (конкретные автотесты) — не согласован.
- Ручная загрузка примеров стиля vs автоматический импорт из профилей — какой вариант в приоритете для подпроекта 2.
- Модерация сгенерированного контента — минимальный объём проверки для MVP, до появления продающих форматов.
- Точные метрики успеха MVP-1 (раздел 7) — требуют подтверждения, сейчас это предложенные ориентиры, не согласованные цифры.
