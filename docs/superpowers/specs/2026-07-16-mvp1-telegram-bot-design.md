# Дизайн: MVP-1 — текстовое ядро ИИ-агента для соцсетей

## 0. Разбиение продукта на подпроекты

Полный продукт из `technical-specification.md` слишком велик для одной спеки/реализации. Разбит на 6 подпроектов с зависимостями:

1. **Текстовое ядро (MVP-1)** — ссылка/голос → транскрибация → варианты текста для Telegram и VK. Ни от чего не зависит.
2. **Анализ стиля автора** — свои образцы + импорт из профилей. Зависит от (1).
3. **Работа с фото/видео** — посты/подписи/хэштеги/музыка по медиа. Зависит от (1).
4. **Веб-приложение** — личный кабинет, история, экспорт. Зависит от (1).
5. **Согласование/коллаборация** — статусы, кнопки, уведомления. Зависит от (1), отчасти (4).
6. **Лидогенерация и самопродвижение** — поиск намерения в чатах, черновики с подтверждением человеком. Технически независим, юридически чувствителен (152-ФЗ, правила Telegram) — реализуется последним, с человеком в контуре.

Порядок реализации: 1 → (2, 3) → 4 → 5 → 6. Каждый подпроект получает свою спеку и план.

## 1. MVP-1: технологический стек и архитектура

### Ограничения, определившие решения
- Владелец продукта не программист.
- Оплата только в рублях, иностранных карт нет.
- На старте бот нужен владельцу и нескольким знакомым (не публичный запуск).
- Интерфейс бота и генерация контента — на 4 языках сразу: русский, английский, вьетнамский, китайский.

### Стек
- **Язык/фреймворк:** Python + aiogram, режим long polling (без домена/SSL).
- **Процесс:** автоперезапуск (systemd/Docker restart policy) + уведомление владельцу в личку при падении бота — иначе самый вероятный сбой останется незамеченным.
- **ИИ:** единый российский прокси-сервис (типа vsegpt.ru) — доступ к Claude/GPT-класс моделям и к Whisper, оплата в рублях. Выбран вместо прямых аккаунтов Anthropic/OpenAI, так как прямая оплата недоступна, а гео-ограничения создают риск блокировки аккаунта, завязанного на карту стороннего человека.
- **AI Gateway:** все вызовы к ИИ-провайдеру спрятаны за внутренним интерфейсом (`generate_text()`, `transcribe()`), а не разбросаны по коду — замена провайдера в будущем должна быть заменой конфигурации, а не переписыванием бота.
- **Извлечение текста по ссылке:** библиотека trafilatura — работает с кириллицей, вьетнамским (латиница с диакритикой) и китайским (эвристики по структуре HTML, а не по токенизации слов).
- **Хранилище:** SQLite — белый список Telegram ID, языковая настройка пользователя (интерфейс отдельно от языка контента), счётчик запросов с дневным/месячным лимитом на человека (защита от перерасхода бюджета на ИИ одним пользователем). Полноценная история контента — вне MVP-1 (см. подпроект 4).
- **Хостинг:** Timeweb Cloud (российский облачный провайдер, оплата картой РФ).
- **Секреты:** токен бота и ключ ИИ-прокси — в `.env`, не в репозитории.

### Локализация
- Язык интерфейса (кнопки, сообщения, ошибки): команда `/language` с инлайн-кнопками RU/EN/VI/ZH; по умолчанию — `language_code` из Telegram при первом обращении. Строки интерфейса — в отдельных словарях по языкам, не в коде обработчиков.
- Язык генерации контента — независимая настройка от языка интерфейса (можно писать посты на языке, отличном от интерфейса); по умолчанию равен языку интерфейса, изменяется пользователем при запросе.
- Язык входного контента (ссылка/голос) определяется отдельно, для целей извлечения/транскрибации — не путать с языком интерфейса.

### Компоненты
1. **Bot Handler** — приём сообщений, проверка белого списка и лимита запросов, определение типа входа (ссылка/голос/текст).
2. **Input Processor** — голос → расшифровка через AI Gateway с показом текста пользователю для проверки/правки перед генерацией (это же снижает риск ошибок распознавания вьетнамской речи, см. ниже); ссылка → извлечение статьи через trafilatura; текст → используется как есть.
3. **AI Gateway** — единая точка вызова ИИ-прокси (транскрибация и генерация текста), см. выше.
4. **Content Generator** — промпт с учётом целевых площадок (Telegram, VK) и выбранного языка контента → несколько вариантов текста.
5. **Localization** — словари строк интерфейса на 4 языках, хранение выбранного языка пользователя.
6. **Output Formatter** — форматирование (разметка, эмодзи) и отправка вариантов с инлайн-кнопками «Ещё вариант» / «Короче».

### Тестирование
- Автотесты пишет разработчик (Claude) в ходе реализации.
- Дополнительно — простой чек-лист ручной проверки для владельца перед каждым обновлением: одна ссылка, одно голосовое, один пользователь не из белого списка.

## 2. Проверка архитектуры (независимая критика)

Дизайн раздела 1 прогнан через два независимых агента — фактчекинг и архитектурную критику — прежде чем считаться финальным.

**Фактические ограничения продукта (зафиксированы, не блокируют MVP-1):**
- Telegram заблокирован в материковом Китае (с 2015 г., доступ только через VPN, с апреля 2026 провайдеры КНР дополнительно блокируют сам VPN-трафик) — бот не достучится до пользователей физически в материковом Китае без VPN на их стороне. Актуально при реальном запуске на китайскую аудиторию, не на этапе теста с друзьями.
- VK практически не используется во Вьетнаме и Китае — вариант текста «под VK» этим пользователям будет просто неактуален, но это не ломает функциональность (они используют вариант «под Telegram»).
- Распознавание вьетнамской речи (Whisper) даёт заметно больше ошибок, чем русской/английской/китайской (тональный, малоресурсный язык). Митигировано тем, что пользователь видит и может поправить расшифровку перед генерацией контента — уже заложено в Input Processor.

**Архитектурные доработки, включённые в раздел 1:**
- AI Gateway как прослойка над ИИ-провайдером (защита от единой точки отказа на уровне кода).
- Автоперезапуск процесса + уведомление владельцу при падении бота.
- Лимит запросов на пользователя поверх белого списка (защита от перерасхода бюджета).
- `/language` + `language_code` по умолчанию вместо детекции языка интерфейса по содержимому сообщений (ненадёжно).
- Ручной чек-лист проверки вместо расчёта на то, что не-программист сам отладит упавший процесс.
