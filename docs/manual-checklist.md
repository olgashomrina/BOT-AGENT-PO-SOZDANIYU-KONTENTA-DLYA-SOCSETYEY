# Ручной чек-лист владельца

Этот файл — инструкции для владельца бота (не программиста), которые нельзя
проверить автоматическими тестами. Написан простым языком, без технического
жаргона там, где это возможно.

## Как добавить пользователя

В MVP-1 нет экрана администратора внутри Telegram — бот пускает только тех,
чей Telegram ID добавлен в специальный список ("вайтлист") прямо в файле
базы данных. Добавить туда себя или знакомого можно вручную, один раз для
каждого человека, следующим способом.

### Шаг 1. Узнать Telegram ID человека

Telegram ID — это число, уникальное для каждого аккаунта Telegram (не
username, не номер телефона). Чтобы узнать его:

1. Откройте Telegram и найдите бота с именем `@userinfobot`.
2. Нажмите "Старт" (Start).
3. Бот пришлёт в ответ несколько строк, среди них — `Id: 123456789`
   (число будет другое). Это и есть нужный Telegram ID.

Попросите каждого знакомого, которого хотите добавить, сделать то же самое
и прислать вам своё число.

### Шаг 2. Открыть терминал в папке проекта

1. Откройте PowerShell (или Git Bash) в папке проекта — там, где лежит папка
   `bot` и файл `requirements.txt`.
2. Убедитесь, что виртуальное окружение установлено (это делалось один раз
   при первой настройке бота, см. `README.md`).

### Шаг 3. Выполнить одну команду добавления

Выполните команду ниже, заменив `123456789` на настоящий Telegram ID
человека, которого добавляете. Если в файле `.env` для `DB_PATH` указано
что-то отличное от `bot.db`, замените `bot.db` в команде на то же значение.

PowerShell:

```powershell
.venv\Scripts\python.exe -c "from bot.storage.db import init_db; from bot.storage.whitelist import add_user; init_db('bot.db'); add_user('bot.db', 123456789); print('Готово: пользователь добавлен')"
```

Если команда вывела строку `Готово: пользователь добавлен` — человек уже
может пользоваться ботом. Файл базы данных (`bot.db`) при этом создаётся
автоматически, если его ещё не было.

Чтобы добавить ещё одного человека — повторите Шаг 3 с его Telegram ID.
Повторный запуск с уже добавленным ID ничего не сломает.

### Как убрать доступ у пользователя

Если нужно забрать доступ у кого-то (например, человек больше не тестирует
бота), выполните похожую команду, заменив `add_user` на `remove_user`:

```powershell
.venv\Scripts\python.exe -c "from bot.storage.whitelist import remove_user; remove_user('bot.db', 123456789); print('Готово: доступ убран')"
```

### Важно знать

- Полная история переписки и сгенерированного контента в MVP-1 **не
  сохраняется** — база данных хранит только список разрешённых пользователей,
  их языковые настройки и счётчик количества запросов (для дневного и
  месячного лимита). Ничего больше.
- Резервных копий файла базы данных (`bot.db`) пока нет — это осознанное
  решение для MVP-1, поскольку в базе нет ничего критически важного (нет
  истории контента, которую жалко потерять). Вопрос вернётся, когда появится
  сохранение истории (уровень Pro, вне MVP-1).
