# Ручной чек-лист владельца

Этот файл — инструкции для владельца бота (не программиста), которые нельзя
проверить автоматическими тестами. Написан простым языком, без технического
жаргона там, где это возможно.

## Как добавить пользователя

В MVP-1 нет экрана администратора внутри Telegram — бот пускает только тех,
чей Telegram ID добавлен в специальный список ("вайтлист") прямо в файле
базы данных. Добавить туда себя или знакомого можно вручную, один раз для
каждого человека, следующим способом.

### Шаг 1. Узнать Telegram ID человека

Telegram ID — это число, уникальное для каждого аккаунта Telegram (не
username, не номер телефона). Чтобы узнать его:

1. Откройте Telegram и найдите бота с именем `@userinfobot`.
2. Нажмите "Старт" (Start).
3. Бот пришлёт в ответ несколько строк, среди них — `Id: 123456789`
   (число будет другое). Это и есть нужный Telegram ID.

Попросите каждого знакомого, которого хотите добавить, сделать то же самое
и прислать вам своё число.

### Шаг 2. Открыть терминал в папке проекта

1. Откройте PowerShell (или Git Bash) в папке проекта — там, где лежит папка
   `bot` и файл `requirements.txt`.
2. Убедитесь, что виртуальное окружение установлено (это делалось один раз
   при первой настройке бота, см. `README.md`).

### Шаг 3. Выполнить одну команду добавления

Выполните команду ниже, заменив `123456789` на настоящий Telegram ID
человека, которого добавляете. Если в файле `.env` для `DB_PATH` указано
что-то отличное от `bot.db`, замените `bot.db` в команде на то же значение.

PowerShell:

```powershell
.venv\Scripts\python.exe -c "from bot.storage.db import init_db; from bot.storage.whitelist import add_user; init_db('bot.db'); add_user('bot.db', 123456789); print('Готово: пользователь добавлен')"
```

Если команда вывела строку `Готово: пользователь добавлен` — человек уже
может пользоваться ботом. Файл базы данных (`bot.db`) при этом создаётся
автоматически, если его ещё не было.

Чтобы добавить ещё одного человека — повторите Шаг 3 с его Telegram ID.
Повторный запуск с уже добавленным ID ничего не сломает.

### Как убрать доступ у пользователя

Если нужно забрать доступ у кого-то (например, человек больше не тестирует
бота), выполните похожую команду, заменив `add_user` на `remove_user`:

```powershell
.venv\Scripts\python.exe -c "from bot.storage.whitelist import remove_user; remove_user('bot.db', 123456789); print('Готово: доступ убран')"
```

### Важно знать

- Полная история переписки и сгенерированного контента в MVP-1 **не
  сохраняется** — база данных хранит только список разрешённых пользователей,
  их языковые настройки и счётчик количества запросов (для дневного и
  месячного лимита). Ничего больше.
- Резервных копий файла базы данных (`bot.db`) пока нет — это осознанное
  решение для MVP-1, поскольку в базе нет ничего критически важного (нет
  истории контента, которую жалко потерять). Вопрос вернётся, когда появится
  сохранение истории (уровень Pro, вне MVP-1).

## Проверка перед каждым обновлением бота

Автоматические тесты (их запускает программист) проверяют код, но не то, как
бот реально ведёт себя в Telegram. Поэтому перед тем как выпускать
обновление (или после переустановки бота на сервере), стоит своими руками
проверить три вещи: ссылку, голосовое сообщение и что чужого человека бот не
пускает. Это займёт 5–10 минут.

Вам понадобится:
- Открытый Telegram с аккаунтом, который уже добавлен в вайтлист (см. раздел
  "Как добавить пользователя" выше) — назовём его "ваш основной аккаунт".
- Желательно — второй Telegram-аккаунт (свой запасной или знакомого), который
  **не** добавлен в вайтлист, для последнего пункта. Если второго аккаунта
  нет — этот пункт можно попросить проверить знакомого.

### Проверка 1. Одна ссылка

1. С вашего основного аккаунта откройте диалог с ботом.
2. Отправьте боту ссылку на любую обычную статью или новость, например:
   `https://ru.wikipedia.org/wiki/Кошка`
3. Подождите до минуты.

Что должно произойти:
- Бот пришлёт несколько вариантов текста поста (для Telegram и для VK), под
  каждым — кнопки вроде "Ещё вариант" и "Короче".

Если что-то не так:
- Бот молчит дольше 1–2 минут — вероятно, бот не запущен или не отвечает на
  сервере (см. `deploy/README-deploy.md`, если он уже готов, или сообщите
  программисту).
- Бот прислал сообщение вида *"Не получилось извлечь текст по этой
  ссылке..."* — это нормальное поведение для защищённых сайтов, но попробуйте
  ещё раз с другой ссылкой (например, с Wikipedia), чтобы убедиться, что дело
  не в самом боте.
- Бот прислал сообщение вида *"Что-то пошло не так на нашей стороне..."* —
  это сигнал реальной проблемы, стоит сообщить программисту.

### Проверка 2. Одно голосовое сообщение

1. С того же аккаунта отправьте боту голосовое сообщение (10–20 секунд,
   на русском языке, скажите пару обычных предложений).
2. Подождите до минуты.

Что должно произойти:
- Бот пришлёт текст того, что он услышал, и спросит "Всё верно?" с кнопками
  "Всё верно" / "Изменить".
- Нажмите "Всё верно" — бот должен прислать несколько вариантов поста, как и
  в проверке со ссылкой.

Если что-то не так:
- Расшифровка сильно не совпадает с тем, что вы сказали — попробуйте ещё раз,
  говоря чуть медленнее и в тихом месте. Если и это не помогает — сообщите
  программисту.
- Бот прислал сообщение вида *"Не получилось распознать голосовое
  сообщение..."* — попробуйте отправить голосовое ещё раз. Если повторяется
  каждый раз — сообщите программисту.

### Проверка 3. Пользователь не из вайтлиста

1. С аккаунта, который **не** добавлен в вайтлист, откройте диалог с ботом и
   отправьте любое сообщение (например, просто слово "привет").

Что должно произойти:
- Бот ответит примерно так: *"Доступ к этому боту сейчас ограничен списком
  приглашённых пользователей. Если вы считаете, что должны иметь доступ,
  свяжитесь с владельцем бота."*
- Бот **не** должен начать генерировать посты для этого аккаунта.

Если что-то не так:
- Бот сгенерировал пост для аккаунта не из вайтлиста — это серьёзная
  проблема (означает, что доступ никак не ограничен), нужно сразу сообщить
  программисту и не публиковать обновление.
- Бот вообще не ответил — как и в проверке 1, проверьте, что бот запущен.

### Что делать по итогам

- Если все три проверки прошли как описано — обновление можно считать
  готовым к использованию.
- Если хотя бы одна проверка дала "серьёзную проблему" (бот падает, пускает
  чужих, ничего не отвечает) — обновление откладывается до исправления.
- Мелкие неточности (например, расшифровка голоса не идеальна) — это не
  повод останавливать обновление, но стоит отметить их для программиста.
