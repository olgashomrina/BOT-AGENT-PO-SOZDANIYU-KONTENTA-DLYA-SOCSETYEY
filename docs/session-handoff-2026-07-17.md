## Обновление — продолжение разговора (следующая сессия)

### Вопрос: OpenRouter вместо vsegpt.ru?

Владелец спросил, можно ли заменить `vsegpt.ru` на OpenRouter. Ответ: это разные категории сервисов, не взаимозаменяемые с Timeweb Cloud (Timeweb — где бот физически работает; vsegpt.ru/OpenRouter — откуда бот берёт ИИ-модель для генерации текста и расшифровки голоса). Решили **остаться на vsegpt.ru**, потому что OpenRouter, насколько известно, не даёт доступа к распознаванию речи (Whisper/транскрипция), а расшифровка голосовых — ключевая часть продукта (Фаза 5). Архитектура бота (`AI_PROXY_BASE_URL`/`AI_GATEWAY_PROVIDER`/`AI_GATEWAY_TEXT_MODEL` в `.env`) и так позволяет сменить провайдера одной строкой в конфиге при необходимости в будущем — код для этого менять не нужно (см. Фазу 3 в `Plan.md`).

Подтверждено: **vsegpt.ru — платный сервис**, отдельный от Timeweb Cloud. Когда дойдёт до реального использования, нужно будет: зарегистрироваться на vsegpt.ru → получить `AI_PROXY_API_KEY` → пополнить баланс в рублях (списывается по факту использования, не абонентка).

### Вопрос: нужен ли команде общий номер телефона / Telegram-аккаунт для управления ботом?

Владелец работает командой из нескольких человек и спросил, не завести ли один общий номер для управления ботом. Ответ — **не нужно**, в проекте уже заложена ролевая модель под это:

1. **Пользоваться ботом** (отправлять ссылки/голос) — каждый под своим личным Telegram-аккаунтом. Их ID просто добавляются в вайтлист (Фаза 1, `bot/storage/whitelist.py`) — он и создан для списка из нескольких разрешённых пользователей.
2. **Уведомления о падении бота** (`OWNER_CHAT_ID` в `.env`) — вместо личного чата одного человека можно указать ID **групповового чата в Telegram** (у групп ID отрицательный, код это уже поддерживает без изменений) — тогда уведомление получит вся команда разом.
3. **Администрирование в @BotFather** (смена описания/команд бота) — единственное место, где технически привязка к одному Telegram-аккаунту. Если нужно, чтобы несколько людей могли туда заходить — Telegram позволяет подключать новые устройства к тому же аккаунту через QR-код (Настройки → Устройства → «Подключить устройство»), без ввода SMS-кода каждый раз и без физического доступа к SIM-карте. Минус: с точки зрения Telegram все, кто залогинен, — «один и тот же человек», не различить, кто что сделал. Альтернатива — завести отдельный служебный аккаунт под администрирование, не смешивая с личными.
4. **Доступ к Timeweb Cloud и vsegpt.ru** — обычные веб-логины, не связаны с Telegram; удобнее хранить в общем менеджере паролей команды (Bitwarden, 1Password и т.п.).

Рекомендация: не заводить общий номер; использовать личные аккаунты + вайтлист для использования бота, групповой чат для уведомлений, и при необходимости отдельный служебный аккаунт только под @BotFather.

## Обновление 2026-07-20 — деплой завершён, MVP-1 в реальной эксплуатации

Сервер на Timeweb Cloud (`147.45.175.189`) поднят и работает. Ассистент подключался напрямую по SSH (ключ создан совместно с владельцем в ходе сессии) и выполнял почти все шаги деплоя сам — владелец печатал/вставлял в терминал только секреты (токен бота, ключ vsegpt.ru), следуя инструкции «собрать строку в Блокноте → вставить одним действием», чтобы секреты не попадали в чат.

**По пути возникли и были устранены:**
- Несколько раз путался токен бота (опечатки при копировании, случайно попал в чат — тогда сразу говорили заменить через Revoke в @BotFather).
- Дублирующиеся строки в `.env` из-за повторных `echo >> .env` — решено скриптом, который пересобирает файл начисто, беря последнее значение по каждому ключу.
- Один раз ассистент сама по ошибке вывела секреты в свой лог при неправильно составленной проверочной команде (`grep` без `-c`) — сразу сказано об этом прямо, токен и ключ пересозданы.
- `systemctl kill` не перезапускает бота (SIGTERM = чистое завершение, а `Restart=on-failure` на это не реагирует) — это нормально, не баг. Реальный автоперезапуск и уведомление владельцу проверены отдельно: временно внедрён `raise RuntimeError(...)` прямо в `bot/main.py` на сервере (не в git), перезапущен сервис, подтверждён CRITICAL-лог + автоперезапуск + получено уведомление в Telegram, после чего код возвращён к чистой версии из git.
- Реальный смоук-тест выявил 2 настоящих интеграционных бага, которых не было видно в тестах с моками: (1) `openai/whisper-1` — неверный ID модели для vsegpt.ru, правильно `stt-openai/whisper-1` (у vsegpt.ru модели распознавания речи с префиксом `stt-`); (2) ошибки 4xx логировались без текста ответа сервиса — добавлено логирование тела ответа.
- По просьбе владельца после первого успешного теста — добавлена инструкция на тон/юмор в промпт генерации, т.к. тексты звучали слишком сухо/корпоративно.
- Устаревший текст приветствия `/start` («работает как заглушка») заменён на актуальный на всех 4 языках.
