# Бот-агент по созданию контента для соцсетей

Telegram-бот, который превращает ссылку на статью, голосовое сообщение или
текст в готовые варианты постов для Telegram и VK.

Эта инструкция написана для человека без опыта программирования. Следуйте
шагам по порядку — ничего дополнительного знать не нужно.

## Что понадобится перед началом

- Компьютер с Windows.
- Токен Telegram-бота (получить у @BotFather, см. шаг 4).
- Ключ доступа к ИИ-сервису (выдаётся отдельно, спросите у того, кто
  настраивал проект).

## Шаг 1. Установить Python

1. Откройте страницу https://www.python.org/downloads/ и скачайте последнюю
   версию Python для Windows.
2. Запустите установщик. **Обязательно** поставьте галочку
   «Add python.exe to PATH» внизу первого экрана установки.
3. Нажмите «Install Now» и дождитесь завершения.
4. Проверьте установку: откройте PowerShell (найдите в меню Пуск) и введите:

   ```
   python --version
   ```

   Если в ответ показалась версия Python (например, `Python 3.13.0`) —
   всё установлено верно.

## Шаг 2. Скачать проект (clone)

Если у вас уже есть папка с проектом — пропустите этот шаг.

1. Установите Git: https://git-scm.com/download/win (можно оставить все
   настройки по умолчанию).
2. В PowerShell перейдите в папку, куда хотите скачать проект, и выполните:

   ```
   git clone <ссылка-на-репозиторий>
   cd "БОТ-АГЕНТ ПО СОЗДАНИЮ КОНТЕНТА ДЛЯ СОЦСЕТЕЙ"
   ```

## Шаг 3. Установить зависимости

Находясь в папке проекта, выполните в PowerShell:

```
pip install -r requirements.txt
```

Это установит всё необходимое для работы бота. Команда должна завершиться
без красных строк с текстом `ERROR`.

## Шаг 4. Получить токен бота и ключ ИИ-сервиса

1. Откройте Telegram, найдите бота **@BotFather**.
2. Напишите ему `/newbot` и следуйте подсказкам (укажите имя и username
   будущего бота).
3. В ответ BotFather пришлёт токен — длинную строку вида
   `123456789:AAExampleTokenValue`. Сохраните её, она понадобится дальше.
4. Ключ доступа к ИИ-сервису (AI-прокси) выдаётся отдельно — узнайте его
   у того, кто настраивал оплату сервиса.
5. Узнайте свой Telegram chat id: напишите боту **@userinfobot**, он пришлёт
   ваш числовой ID.

## Шаг 5. Заполнить файл .env

1. В папке проекта найдите файл `.env.example`.
2. Сделайте его копию и назовите новую копию `.env` (без `.example` на
   конце).
3. Откройте `.env` в любом текстовом редакторе (например, Блокнот) и
   заполните значения после каждого знака `=` — без пробелов и без кавычек:

   ```
   BOT_TOKEN=токен_от_BotFather
   AI_PROXY_API_KEY=ваш_ключ_ии_сервиса
   OWNER_CHAT_ID=ваш_числовой_chat_id
   ```

4. Остальные строки (`DAILY_LIMIT`, `MONTHLY_LIMIT`, `DB_PATH`, `LOG_LEVEL`)
   можно оставить как есть — там уже подставлены разумные значения по
   умолчанию.
5. Сохраните файл.

Файл `.env` содержит секретные данные — никогда не отправляйте его
никому и не выкладывайте в открытый доступ. Он уже настроен так, чтобы не
попадать в Git.

## Шаг 6. Запустить бота

В папке проекта выполните:

```
python -m bot.main
```

Если всё настроено верно, в консоли появится сообщение о запуске, и бот
начнёт отвечать в Telegram. Отправьте ему `/start` — должен прийти
приветственный текст.

Чтобы остановить бота, нажмите `Ctrl + C` в окне PowerShell.

## Проверка, что всё работает правильно (для разработчика)

```
pytest
```

Все тесты должны пройти без ошибок (`passed`, без `failed`).

## Известные ограничения (важно понимать)

- **Локализация интерфейса ≠ выход на рынки Китая и Вьетнама.** Telegram
  заблокирован в материковом Китае (с 2015 года) и во Вьетнаме (с мая 2025
  года). Языки интерфейса RU/EN/VI/ZH облегчают жизнь диаспоре и
  пользователям с VPN, но не открывают массовый локальный рынок этих стран.
- Это MVP-1 — минимальная версия. На данном этапе (скелет проекта) бот умеет
  только отвечать заглушкой на `/start`. Генерация контента, вайтлист,
  лимиты и локализация появятся в следующих фазах разработки (см.
  `Plan.md`).

## Частые проблемы

- **`python` не распознаётся как команда** — Python установлен без галочки
  «Add to PATH». Переустановите Python и поставьте эту галочку (Шаг 1).
- **Бот не отвечает на `/start`** — проверьте, что в `.env` указан
  правильный `BOT_TOKEN` без лишних пробелов, и что бот запущен (в консоли
  нет сообщений об ошибке).
- **Ошибка про отсутствующую переменную окружения** — откройте `.env` и
  убедитесь, что заполнены все обязательные поля: `BOT_TOKEN`,
  `AI_PROXY_API_KEY`, `OWNER_CHAT_ID`.
