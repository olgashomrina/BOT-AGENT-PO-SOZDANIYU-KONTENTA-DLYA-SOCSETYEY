# Пользовательские сценарии для ИИ-агента контента

## Сценарий 1: Быстрое создание контента из ссылки
1. Пользователь (маркетолог, предприниматель, SMM‑специалист) заходит в веб-интерфейс или открывает Telegram-бота.
2. Загружает ссылку на статью, пост, продуктовую страницу или материал конкурента.
3. Бот анализирует ссылку, извлекает ключевые идеи и структуру.
4. Пользователь выбирает площадки (Telegram, ВКонтакте, сторис, email) для генерации.
5. Бот выдает несколько вариантов текста, готовых для публикации, с эмодзи, CTA и рекомендациями по визуалам.
6. Пользователь копирует выбранный вариант или экспортирует его в файл.

## Сценарий 2: Голосовой ввод и транскрибатор
1. Пользователь отправляет в Telegram-бот голосовое сообщение или загружает аудио-файл.
2. Бот автоматически транскрибирует аудио и показывает расшифровку.
3. Пользователь подтверждает или редактирует транскрипт при необходимости.
4. Бот на основе текста генерирует контент для выбранных площадок.
5. Пользователь получает готовый пакет текстов и промптов для визуалов.

## Сценарий 3: Согласование и коллаборация
1. Владелец контента или менеджер создает вариант текста через бота.
2. Бот предлагает кнопки согласования: "Утвердить", "Запросить правки", "Отправить коллеге".
3. Другой пользователь (коллега, заказчик, менеджер) получает уведомление или ссылку на историю контента.
4. Он нажимает кнопку согласования и оставляет комментарий, если нужно.
5. Бот фиксирует результат: утверждено, отправлено на правку или отклонено.
6. При необходимости бот генерирует новую версию с учётом комментариев.

## Сценарий 4: Адаптация стиля пользователя
1. Пользователь добавляет свои образцы текста или разрешает автоматический импорт из публичных профилей (Telegram, VK, Instagram, сайт/блог).
2. Бот анализирует стиль, тональность, ключевые фразы и предпочтения формата.
3. При генерации нового контента бот предлагает варианты, отражающие стиль пользователя.
4. Пользователь может выбрать один из вариантов и дополнительно скорректировать тон.

## Сценарий 5: Ежедневная работа с контент-пулом
1. Пользователь открывает личный кабинет на веб-платформе.
2. В разделе истории видит все ранее созданные тексты, транскрипты и согласования.
3. Пользователь дублирует или обновляет старый вариант для новой темы.
4. Бот помогает превратить идею в новый цикл контента, предлагая адаптацию под разные каналы.
5. Результат экспортируется в CSV, DOCX или напрямую в интегрированные соцсети/инструменты.

## Сценарий 6: Реклама и офферы
1. Пользователь выбирает цель контента: лид-магнит, продажа консультации, запуск курса, промо пост.
2. Бот предлагает релевантные CTA и варианты подачи: продающий, экспертный, вовлекающий.
3. Пользователь получает сразу несколько вариантов, готовых для разных площадок.
4. Бот также предлагает промпты для создания визуала и рекомендации инструментов для генерации медиа.

## Сценарий 7: Работа в команде над кампанией
1. Руководитель проекта создает черновой контент и помечает его как "на согласование".
2. Коллеги получают доступ к цепочке утверждений и могут работать с кнопками "Согласовать" / "Потребуются правки".
3. Все действия сохраняются в истории контента для аудита и повторного использования.
4. После согласования контент отправляется в Export-поток или публикуется автоматически.
