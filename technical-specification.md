# Техническое задание: ИИ-агент для создания контента в соцсетях

**Источник:** последний коммит из папки `my-lending-test` (lending-проект), добавлен `idea.md` 14.07.2026.

## 1. Цель
Создать продукт, который по одной ссылке или голосовому сообщению генерирует готовый пакет контента для Telegram, ВКонтакте и сторис. Продукт должен помогать маркетологам, предпринимателям, SMM‑специалистам, экспертам и другим специалистам быстро получить варианты текстов, визуалы и промпты для генерации медиа, не ограничиваясь идеями для лендингов.

## 2. Задачи MVP
- Приём исходного ввода пользователя:
  - ссылка на материал,
  - голосовое сообщение (возможность расшифровки и извлечения смысла из неструктурированного текста).
- Встроенный транскрибатор для голосовых сообщений и загруженных аудио-файлов.
- Генерация нескольких вариантов текста под каждую площадку:
  - Telegram,
  - ВКонтакте,
  - сторис.
- Формирование готовых текстов с разметкой, эмодзи и CTA.
- Генерация рекомендаций по визуалам:
  - готовые идеи для сторис,
  - промпты для генерации фото/видео.
- Учет стиля пользователя из его прошлых постов, чтобы выдача звучала его голосом.
- Доступ через Telegram-бота и веб-приложение с личным кабинетом:
  - история постов,
  - редактирование,
  - управление контентом.

## 3. Аудитория
- Перегруженные маркетологи.
- Предприниматели и владельцы малого и среднего бизнеса.
- SMM‑специалисты и контент-менеджеры.
- Эксперты, коучи и консультанты.
- Владелец продуктов и услуг, продающих в соцсетях.
- Специалисты из разных направлений бизнеса, которым нужен быстрый и умный помощник для создания маркетингового, экспертного и продающего контента.

## 4. Источники информации
- Исходный ввод пользователя: ссылка или голосовое сообщение.
- Дополнительный поиск по теме в интернете:
  - похожие посты конкурентов,
  - научные статьи,
  - исследования,
  - книги,
  - интервью,
  - подкасты,
  - релевантные экспертные мнения.
- Прошлые публикации пользователя для адаптации стиля.

## 5. Функции и результаты
- Набор вариантов текстов на выбор.
- Отдельные версии для Telegram, ВКонтакте и сторис.
- Готовые визуальные идеи и сторис-контент.
- Промпты для генераторов фото/видео.
- Рекомендации по инструментам для создания медиа.
- Дополнительные форматы: опросы, тесты, цепляющие фразы, продающие тексты и офферы.
- CTA-кнопки/призывы, ведущие на продукт или лид-магнит.

## 6. Вопросы для уточнения
- Нужен ли отдельный шаг, где пользователь заранее загружает примеры своего стиля?
- Нужен ли импорт контента из публичных профилей пользователя автоматически?
- Какие именно продукты/предложения будут продвигаться через тексты?
- Как устроена монетизация лид-магнитов и какие варианты CTA требуются?
- Предполагается ли поддержка других площадок в будущем (Instagram, LinkedIn, Facebook и др.)?

## 6a. Ответы на уточняющие вопросы
- Предпочтителен гибридный вариант: и загрузка собственных примеров, и автоматический импорт из публичных профилей.
- Оба варианта должны быть доступны пользователю.
- В текстах будут продвигаться все возможные форматы: консультации, онлайн-курсы, марафоны, лид-магниты, товары/продукты и партнёрские офферы.
- Монетизация лид-магнитов будет комбинированной: сбор контактов, прямое предложение платных продуктов и повышение узнаваемости бренда.
- Нужно сразу планировать поддержку других площадок помимо Telegram, ВКонтакте и сторис.
- Целевая аудитория будет смешанной: маркетологи, эксперты, владельцы малого бизнеса и копирайтеры.
- Важно обеспечить всё вместе: скорость, персонализацию и мультиплатформенность.
- В MVP должны быть оба входа: Telegram-бот и веб-форма.
- Пользователи будут генерировать контент ежедневно.
- В MVP должны быть и прямой экспорт/публикация, и возможность копировать/скачивать готовые тексты и промпты.
- Голосовой ввод должен поддерживать и Telegram-голосовые сообщения, и загрузку аудио-файлов.
- Для автоматического анализа стиля нужно использовать все доступные публичные профили: Telegram, VK, Instagram, сайт/блог.

## 7. Предложенная архитектура MVP
1. Входной интерфейс:
   - Telegram-бот;
   - веб-форма.
2. Обработка входных данных:
   - встроенный транскрибатор для голосовых сообщений и аудио-файлов;
   - распознавание речи;
   - извлечение смысла из ссылки или аудио;
   - анализ стиля пользователя.
3. Генерация контента:
   - тексты для нескольких площадок;
   - инструкции для визуалов;
   - описания CTA;
   - кнопки согласования контента для автора и других участников.
4. Хранение и управление:
   - история созданного контента;
   - редактирование и экспорт.

## 8. Критерии успеха
- Пользователь получает рабочий пакет контента за один цикл.
- Тексты звучат персонально и соответствуют заданной нише.
- Система выдаёт готовые CTA и визуальные идеи.
- Пользователь может выбрать из нескольких вариантов.
